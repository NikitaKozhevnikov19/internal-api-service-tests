# Автотесты для внутреннего API сервиса (Internal App)

Данный проект содержит набор интеграционных тестов для проверки ключевых операций API: `LOGIN`, `ACTION` и `LOGOUT`.

## 🛠 Стек технологий
* **Java 17** + **Gradle**
* **JUnit 5** (Тестовый движок)
* **RestAssured** (Клиент для API)
* **WireMock** (Эмуляция внешних зависимостей)
* **Allure Report** (Отчетность с кастомными шаблонами)
* **AssertJ** (Fluent assertions)
* **JSON Schema Validator** (Контрактное тестирование)

## 🚀 Особенности реализации
1. **Интеграционный подход**: Тесты автоматически поднимают **WireMock** сервер на порту `8888` для эмулирования ответов внешнего сервиса (параметр `-Dmock` в приложении).
2. **Контрактное тестирование**: Внедрена проверка ответов по **JSON-схеме** (`schema.json`), что гарантирует стабильность структуры API.
3. **Адаптивная валидация данных**: В ходе тестирования выявлено, что параметр `token` требует строгого Hex32 формата (`0-9A-F`). Генератор данных (`DataGenerator`) настроен в соответствии с этими требованиями.
4. **Кастомная отчетность**: Подключены `.ftl` шаблоны для Allure, что позволяет наглядно просматривать Request/Response прямо в отчете.
5. **Fluent API**: Тесты написаны в "столбик" для максимальной читаемости и поддержки кода.

## Запуск
1. Запустить тестируемое приложение:
   `java -jar -Dsecret=qazWSXedc -Dmock=http://localhost:8888/ internal-0.0.1-SNAPSHOT.jar`
2. Запустить тесты:
   `./gradlew clean test`
3. Сгенерировать отчет:
   `./gradlew allureServe`
